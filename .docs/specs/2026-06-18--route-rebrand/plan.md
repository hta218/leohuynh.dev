# Plan — Route rebrand

## 1. Goal

Replace v4's generic routes with workspace/terminal-flavored ones, matching the
"code studio" identity, **without breaking SEO or deep links**. Home stays the
workspace README.

2026-07-08 SEO rollback: keep the developer-flavored UI label for the writing
section (`logs` / `Log`) but restore blog content's public/canonical URL pattern
to the legacy `/blog` prefix. This avoids an unnecessary Google re-indexing
migration for established blog URLs. See
`handoff--2026-07-08--restore-blog-url-canonical.md` for the implementation
scope.

## 2. Route map

| Old route | New route | Notes |
| --------- | --------- | ----- |
| `/` | `/` | unchanged — workspace README (home) |
| `/about` | `/whoami` | unix-flavored "about me" |
| `/blog` | `/log` | + `/log/[...slug]`, `/log/page/[page]` |
| `/snippets` | `/gists` | + `/gists/[...slug]` |
| `/projects` | `/builds` | |
| `/books` + `/movies` | `/shelf` | **merged** into one page (Reading + Watching) |
| `/tags` | `/topics` | + `/topics/[tag]` |

**Slugs are preserved** — only the section prefix changes (`/blog/foo` →
`/log/foo`). This keeps every deep link redirectable 1:1.

Feeds / data endpoints:
- `/feed.xml` — unchanged (site RSS).
- `/tags/[tag]/feed.xml` → `/topics/[tag]/feed.xml`.
- `/search.json` — unchanged path; update any route strings it emits.

## 3. Redirects (SEO — required)

Add **301** redirects in BOTH `astro.config.mjs` (`redirects`) and `vercel.json`
(the real deploy layer). Keep slugs identical:

```
/about              -> /whoami
/blog               -> /log
/blog/:slug*        -> /log/:slug*
/blog/page/:n       -> /log/page/:n
/snippets           -> /gists
/snippets/:slug*    -> /gists/:slug*
/projects           -> /builds
/books              -> /shelf
/movies             -> /shelf            (optionally /shelf#watching)
/tags               -> /topics
/tags/:tag          -> /topics/:tag
```

- **Protect evergreen traffic**: `/blog/does-promise-all-run-in-parallel-or-sequential`
  → `/log/does-promise-all-run-in-parallel-or-sequential` must 301 cleanly.
- The existing `vercel.json` redirect `/snippets/crawling-goodreads-books-data`
  → `/blog/crawling-goodreads-books-data` must be re-pointed to `/log/...` (or it
  will chain through the new `/blog → /log` rule — verify no redirect loop).
- The legacy `v3.leohuynh.dev` deploy is unaffected (separate project/branch).

## 4. Files & folders touched

Pages (rename/move under `src/pages/`):
- `about.astro` → `whoami.astro`
- `blog/` → `log/` (`index.astro`, `[...slug].astro`, `page/[page].astro`)
- `snippets/` → `gists/` (`index.astro`, `[...slug].astro`)
- `projects.astro` → `builds.astro`
- `books.astro` + `movies.astro` → **new** `shelf.astro` (combined page)
- `tags/` → `topics/` (`index.astro`, `[tag].astro`, `[tag]/feed.xml.ts`)

Wiring / cross-references:
- `src/lib/site.ts` — nav `href`/`title` (Blog→Log, Snippets→Gists, Projects→
  Builds, Books+Movies→Shelf, Tags→Topics, About→whoami).
- `src/components/studio/StudioShell.astro` — the explorer file-tree labels/links
  and tab links must match the new routes.
- Any `layouts/` + `components/` with hardcoded `/blog`, `/snippets`, `/projects`,
  `/books`, `/movies`, `/tags`, `/about` links (grep before/after).
- Canonical URL + OpenGraph + breadcrumb helpers (e.g. `app/seo`/`genPageMetadata`
  equivalents) that build paths.
- `content.config.ts` is unaffected (collections still load from `data/`); only
  the page routes that render them move.

Config:
- `astro.config.mjs` — `redirects` map (§3).
- `vercel.json` — `redirects` (§3); re-point the goodreads redirect.
- Sitemap is generated by `@astrojs/sitemap` — will pick up new routes
  automatically; verify output.

Repo-only: this spec folder.

## 5. `/shelf` merge (the one non-trivial page)

`/books` and `/movies` are separate pages backed by `booksTable` / `moviesTable`.
`/shelf` becomes one page with two sections — **Reading** (books) and **Watching**
(movies) — e.g. in-page tabs or stacked sections. Both data sources are queried
in the single page. Old `/books` and `/movies` 301 to `/shelf`.

## 6. Rollout

1. Move/rename page files + create `shelf.astro`; keep slugs.
2. Update `site.ts` nav + StudioShell explorer/tabs + any hardcoded links.
3. Add redirects in `astro.config.mjs` + `vercel.json`.
4. Update canonical/OG/feeds for new paths.
5. Build; diff route inventory old vs new; verify every old path 301s and no
   redirect loops (esp. the goodreads + promise-all evergreen URLs).
6. Verify sitemap/RSS reflect new routes.

## 7. Acceptance criteria

- New routes serve correctly; nav, explorer tree, and tabs point to them.
- Every old route (`/about`, `/blog*`, `/snippets*`, `/projects`, `/books`,
  `/movies`, `/tags*`) returns a 301 to its new path, slugs preserved.
- Evergreen `/blog/does-promise-all-...` → `/log/...` works; no redirect loops.
- `/shelf` shows both Reading and Watching.
- Canonical URLs, sitemap, and RSS use the new paths.

## 8. Not in scope

- Changing post/snippet **slugs** (preserved).
- The `v3.leohuynh.dev` legacy deploy (untouched).
- v4 → `main`/`www` production cutover (separate effort) — but this rebrand
  should land before that cutover so the public site ships with final URLs.
