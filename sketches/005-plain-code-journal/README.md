## Variant: Plain Code Journal

### Design stance
A very minimal light journal that uses code/editor conventions only as quiet structure: file tabs, gutter numbers, commit metadata, and small status widgets.

### Key choices
- Layout: document-first single column with a compact right rail.
- Typography: sans for reading, monospace for system metadata.
- Color: off-white canvas, black text, one blue accent.
- Interaction: switch between Today / Writing / Code activity panes.

### Trade-offs
- Strong at: simplest, most timeless, least likely to age badly.
- Weak at: may feel too plain unless production polish is excellent.

### Best for
A minimalist personal blog with coder vibe through details rather than big visual gestures.
